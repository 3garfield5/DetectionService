# DetectionService

Обнаружение оставленных объектов в видеопотоке

## Общее описание

DetectionService — прототип системы компьютерного зрения для обнаружения оставленных (брошенных) объектов в видеопотоке.
Проект реализует полный pipeline: обработка видео → детекция и трекинг объектов → формирование события → backend-сервис → веб-интерфейс оператора.

Система ориентирована на мониторинг публичных пространств (аэропорты, вокзалы, метро, учебные корпуса).

---

## Постановка задачи

В общественных местах люди могут оставлять личные вещи, что потенциально представляет угрозу безопасности или мешает эксплуатации пространства.
Задача системы — автоматически выявлять такие объекты в видеопотоке и предоставлять оператору информацию для принятия решения.

---

## Логика работы системы

1. Источник видео: RTSP-поток или видеофайлы (`mp4`)
2. Препроцессинг кадров (приведение к разрешению, буферизация)
3. Детекция объектов (сумки, рюкзаки, чемоданы и т.п.)
4. Трекинг объектов и людей
5. Связывание объекта с владельцем
6. Правило «оставленности»:

   * если связанный человек отсутствует рядом с объектом дольше T секунд
   * или расстояние превышает порог R
     → формируется событие «оставленный объект»
7. Передача события в backend и отображение во frontend-интерфейсе

## Архитектура

### Модуль компьютерного зрения

Функции:

* обработка видеопотока;
* детекция и трекинг объектов;
* сопоставление объекта и человека;
* формирование события «оставленный объект»;
* генерация snapshot (bounding box, confidence).

Стек:

* Python
* Ultralytics YOLO
* OpenCV

---

### Backend

Функции:

* приём и хранение событий;
* API для получения информации об обнаруженных объектах;
* обновление данных в реальном времени (WebSocket).

Стек:

* FastAPI
* SQLAlchemy
* PostgreSQL
* Docker

---

### Frontend

Функции:

* отображение видеопотока с разметкой;
* список событий и их метаданных;
* подтверждение / отклонение событий оператором.

Стек:

* React
* Vite

---

## Цели проекта

* Реализовать работоспособный прототип системы обнаружения оставленных объектов
* Продемонстрировать взаимодействие ML-модуля, backend и frontend
* Обеспечить визуализацию и ручную валидацию событий

---

## Ограничения

* Снижение точности при низком качестве видео и плохом освещении
* Возможные пропуски при окклюзиях объектов
* Для работы в реальном времени требуется оптимизация или GPU
* Ограниченная масштабируемость по числу видеопотоков
